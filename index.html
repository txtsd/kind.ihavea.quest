kind

kind is not discord

a native Discord client built from scratch in C++ and Qt. no Electron. no compromise.

source sponsor

why kind exists

the official Discord client is Electron. it wraps a browser around a website and calls it an app. it uses hundreds of megabytes of memory to do what a native application could do with a fraction of that.

kind is built from scratch as a native application in C++ and Qt 6. the difference is tangible. the UI loads instantly. on first launch, things appear as fast as Discord's servers respond. on every launch after that, everything comes from cache and is validated against the network in the background. it feels as fast as opening a plaintext file.

kind targets Windows, macOS, and Linux.

what's built

  • gateway

    WebSocket connection to Discord with exponential backoff, automatic reconnect, and heartbeat management.

  • REST

    full HTTP client with per-route rate limiting that respects Discord's rate limit headers.

  • cache

    SQLite-backed persistent cache with async reads and writes. bulk loading takes the full database from disk to memory in around 110ms.

  • permissions

    complete Discord permission resolution across roles, channels, and overwrites.

  • message rendering

    composable block renderer that handles embeds, attachments, reactions, stickers, and interactive components.

  • markdown

    full Discord-flavour markdown parser with support for all formatting, mentions, and custom emoji.

  • image cache

    async image loading with memory LRU and disk persistence. images appear instantly on repeat views.

  • unread state

    unread and mute tracking that persists across restarts and reconciles with Discord's server state.

  • direct messages

    full 1-on-1 direct messaging with its own dedicated view and conversation list.

  • 370+ tests

    three-tier test suite covering core logic, GUI components, and full integration tests against a mock Discord server.
